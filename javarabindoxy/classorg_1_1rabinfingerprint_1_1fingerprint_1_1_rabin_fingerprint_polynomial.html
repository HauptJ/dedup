Detailed Description

This class implements Rabin's Fingerprinting scheme using a Polynomial class which handles calculations in the finite field GF(2). This is slower than RabinFingerprintLong, but can support monic irreducible polynomials of nearly any degree.

    Given an n-bit message m_0, ..., m_n-1, we view it as a polynomial of degree
    n-1 over the finite field GF(2).
        m(x) = m_0 + m_1 x + ... + m_n-1 x_n-1
    We then pick a random irreducible polynomial p(x) of degree k over GF(2), and
    we define the fingerprint of m to be
        f(x) = m(x) mod p(x)
    which can be viewed as a polynomial of degree k-1 or as a k-bit number.

Because we are operating in the space defined by mod p(x), we only ever need to store a fingerprint of k bits. All new bits are shifted in and modded with p(x), resulting in a new k-bit number.

    This follows from the fact that given an n-bit message:
        m(x) = m_0 x_n-1 + .... + m_n-2 x + m_n-1;
    and its fingerprint
        f(x) = m(x) mod p(x)
             = r_0 x_k-1 + ... + r_k-2 x + r_k-1
    appending one more bit simplifies to
        f({m(x)*x + m_n}) = {m(x)*x + m_n} mod p(x);

This means that to add one bit, we need merely shift in that bit and re-mod with p(x). Similarly, we can add entire bytes, words, etc, by shifting them in and modding with p(x). This can become untennable and slow if we shift too much at once, as the mod calculation is done with synthetic division and will take on the order of {number of bits shifted in} to complete.

A table lookup method is obviously possible, and that is exactly what we do in RabinFingerprintLong.

synchronized void org.rabinfingerprint.fingerprint.RabinFingerprintPolynomial.popByte ( )

Removes the contribution of the first byte in the byte queue from the fingerprint.

Note that the shift necessary to calculate it's contribution can be sizeable, and the mod calculation will be similarly slow. It is therefore done with the Polynomial to support arbitrary sizes.

Note that despite this massive shift, the fingerprint will still result in a k-bit number at the end of the calculation.
